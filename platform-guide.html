// What we do

Three Pillars

Everything on the platform maps to one of three core activities — Learn, Build, or Connect.

01 — LEARN
Explorer Hub
Structured challenge tracks from curious beginner to lab-certified innovator. Earn badges at every tier.
  • T1 — Explorer challenges (curiosity + experimentation)
  • T2 — Solver challenges (real problems + prototypes)
  • T3 — Architect challenges (systems + innovation)
  • Module completions tracked in real-time
  • Certificates + digital badges on completion
02 — BUILD
Projects
Propose, collaborate, and ship real maker projects with structured proposals, GitHub-linked docs, and team tracking.
  • Structured project proposal framework
  • GitHub README template for documentation
  • Team member roles + supervisor links
  • Project updates + milestone tracking
  • Public showcase-ready project pages
03 — CONNECT
Events
Build Challenges, Maker Meetups, and Tech Tuesdays — structured events that turn ideas into projects and makers into community.
  • Build Challenges (hackathons, makeathons, sprints)
  • Maker Meetups (open showcase + feedback)
  • Tech Tuesdays (weekly spotlight sessions)
  • QR check-in → badge auto-issued
  • Submission portal + project showcase
// 01 — Learn

Explorer Hub

Three progressive challenge tiers. Each has a defined structure, constraints, deliverables, and reflection — not just tutorials.

T1
Explorer
Curiosity + Experimentation
Open to All
Format
Driving Question: How does X affect Y?
Context
Explore a concept through direct hands-on experimentation
Materials
Pre-supplied, constrained kit — no buying required
Output
Simple model, observation, comparison, or explanation
Reflection
What changed? What stayed the same? What's happening?
Skills
Observation, hypothesis, basic measurement
T2
Solver
Problem + Prototype
Requires T1
Format
Design a way to solve X using Y resources
Context
Addresses a real or simulated community need
Constraints
Budget cap, material list, time limit, restricted items
Criteria
3 measurable success criteria defined upfront
Deliverable
Working prototype + test report + reflection
Reflection
What worked? What failed? What would you change?
T3
Architect
System + Innovation
Requires T2
Format
Design Brief: Build a system for a target user/context
Why It Matters
Real-world relevance — must articulate impact
Scope
Multiple subsystems, reliability, safety, usability, scale
Deliverables
Prototype + documentation + testing evidence + review
Review Q
Why this architecture? What are the failure points?
Duration
Days to weeks — full project lifecycle

// 03 — Connect

Events

Three distinct event formats, each with a clear purpose, flow, and set of required features. Not just meetups — structured experiences.

⚡
Build Challenges
Hackathons · Makeathons · Design Sprints

Time-bound innovation events where participants explore ideas, solve problems, and build prototypes in response to a theme or real-world need.

  • Challenge brief page with rules + FAQ
  • Individual + team registration
  • Mentor listing with availability
  • Schedule display (phases + deadlines)
  • Submission portal (linked to GitHub)
  • Project showcase after event
  • Winner upload to platform
  • Admin approval before going live
🤝
Maker Meetups
Open Community Gatherings

Open showcases where makers display products, prototypes, and ongoing work — receive feedback and build collaborations.

  • Meetup listing page
  • Showcase slot booking (with approval)
  • Attendee registration (RSVP)
  • Speaker / presenter application
  • Product demo listing
  • Booking confirmation workflow
  • Session gallery (photos / recap)
  • Invitees → converted to public event
📺
Tech Tuesdays
Recurring Weekly Showcase

Recurring sessions hosted at Param where audiences explore new products, projects, prototypes, and innovations emerging from the ecosystem.

  • Recurring weekly publishing
  • Featured lineup for each session
  • Product spotlight cards
  • Speaker / team profile display
  • RSVP system
  • Archive of past sessions
  • Media, recordings, recap posts
  • Auto-badge on attendance
